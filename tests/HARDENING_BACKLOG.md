# Test Hardening Backlog

Generated by forensic review of all 64 test files under `tests/`.
Each item identifies: file, test function, weakness class, and exact change required.

> **Line-number anchor:** ranges were recorded against commit `d2c397e` (branch `frontend-game-mockup`).
> If a file has been edited since, run `git blame <file>` to relocate the correct lines before implementing.

## Weakness class legend

| Code | Description |
|------|-------------|
| W-MSG | Assertion lacks `msg=`; failure output names no concrete wrong value |
| W-LOOSE | Assertion is trivially satisfiable or weaker than it should be |
| W-EDGE | Missing edge / boundary / inverse case |
| W-INV | Tested positive case has no inverse (opposite value) test |
| W-DISJUNCTION | `or` in assertion lets either half satisfy the test |
| W-INDEX | Index-based list access (`[-1]`, `[0]`) fragile to ordering changes |
| W-SRCCODE | Behavioral contract verified by source-text inspection instead of runtime |
| W-EXACT | Key-set check uses `assertIn` where `assertEqual(set(...), ...)` is possible |
| W-EXCEPT | `assertRaises(Exception)` — too broad, should name the specific exception |
| W-SUBTEST | Loop over cases without `subTest` or `msg=`; first failure swallows the rest |
| W-TIMING | Wall-clock sleep used to assert time-sensitive behavior |
| W-NOGUARD | Test reads a file path that may not exist without a `skipTest`/`assertTrue(exists)` guard |
| W-KEYERR | Dict key access without prior `assertIn` guard; `KeyError` would mask the real failure |
